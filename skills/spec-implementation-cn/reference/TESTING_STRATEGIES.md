# 测试策略

下述为各类测试的目的与粗略的覆盖率目标：

## 单元测试 - 90-100%

由程序员编写，面向程序员，用于在最低层次对系统进行规格化，并作为持续集成的一部分执行。

## 组件测试 - 50%

由 QA 与业务在开发者协助下编写，主要覆盖常规路径与明显的边界情况，验证给定输入的输出符合期望。

### 端到端测试 - 70-100%

由 QA 与业务在开发者协助下编写，“端到端”测试是指你做一个断言，这个断言涉及系统逻辑的一个完整“路径”。也就是说，你启动整个系统，在用户输入的入口点执行某些操作，然后检查系统产生的结果。你不在乎内部如何实现这个目标，你只关心输入和结果。这对于所有测试通常都是真实的，但在这里我们只是在系统输入的最外层进行测试，并检查它产生的最外层结果。

## 集成测试 - 20%

通常由系统架构师或首席设计师编写，确保组件正确连接并清晰通信。

## 系统测试 - 10%

通常由系统架构师与技术负责人编写，测试系统被正确地构建。

## 手工探索性测试 - 5%

由测试人员（或任何人）进行，探索系统在人工操作下的表现，并尽可能创造性地发现尽量多的“怪异”行为。
